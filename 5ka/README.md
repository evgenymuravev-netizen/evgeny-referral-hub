# Пятёрочка · АСУЗ — прототип

«Moneyball для розницы»: лендинг с историей продукта, который ведёт в кликабельный
прототип системы управления магазином (АСУЗ — автоматизированная система управления
задачами). Собран по приложенным скриншотам рабочей системы.

## Как открыть

Откройте `index.html` в любом браузере (или по адресу GitHub Pages).
Зависимостей и сборки нет — чистый HTML/CSS/JS.

- `index.html` — **лендинг**: история в духе «Moneyball» (проблема → инсайт → система → показатели).
- `app.html` — **прототип системы** (SPA с хэш-роутингом).

## Карта прототипа

| Экран | Маршрут | Источник |
|---|---|---|
| Главная (сводка директора) | `app.html#/home` | синтез |
| Аналитика · Карточки | `app.html#/analytics` | скриншоты «Аналитика» (РТО, план, потери, CSI, NPS) |
| Аналитика · Дашборд | тот же экран, переключатель | реконструкция файла «Дашборд» |
| Задачи (они же отклонения) | `app.html#/tasks` | скриншот «Активность» + «Постановка задач» |
| Постановка задач / чек-лист | `app.html#/task-create` | скриншот формы постановки |
| АСУЗ · структура скриптов | `app.html#/scripts` | скриншот таблицы «структура скриптов» |
| Операционная модель / проблемы | `app.html#/opmodel` | реконструкция по брифу |

## Что откуда

- **Цифры на экранах** (РТО 128 503 ₽, план 103,58 %, потери 9,18 %, NPS 30 %, счётчики
  типов задач, мартовский магазин 18044 и т. д.) — взяты прямо со скриншотов.
- **Структура скриптов АСУЗ** (14 блоков, ведущие, тексты скриптов) — расшифрована со
  скриншота таблицы.
- **Задачи-отклонения** (ИНФОЦЕНТР/ПРОДАЖИ/ПОТЕРИ, каскадирование, статусы) — со скриншота
  списка задач; активные отклонения достроены и привязаны к скриптам-источникам.
- **Реконструкция** (помечена ⚠️ в интерфейсе): история «Moneyball», «альтернативный
  дашборд» (файл «Дашборд.pdf»), «семь типов процессов Х5» и ключевые проблемы опер-модели
  («Опер-модель.pptx»). Исходные .pptx/.pdf были недоступны в облачной среде сборки —
  содержание восстановлено по брифу и скриптовым блокам.

## Структура файлов

```
5ka/
├── index.html          лендинг (история Moneyball)
├── app.html            оболочка прототипа (sidebar + topbar + роутер)
└── assets/
    ├── data.js         все данные: магазины, задачи, скрипты, процессы (window.DB)
    ├── app.css         дизайн-система и стили приложения
    └── app.js          SPA-роутер и рендер представлений
```

## Ключевая идея

Каждое **отклонение**, найденное скриптом АСУЗ, превращается в **задачу** директору с
понятным «что сделать» и привязкой к скрипту-источнику. Вместо потока отчётов —
одно приоритизированное действие. Контур замыкается: выполнил → показатель изменился.
